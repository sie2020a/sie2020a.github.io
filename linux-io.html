Linux 標準入出力・リダイレクト・パイプ

  • 標準入力 / 標準出力 / 標準エラー出力
  • リダイレクト：>, >>, 2>, 2>&1
  • パイプ：|（前のコマンドの出力→次のコマンドの入力）
  • コマンド：cat, less, head, tail, grep

1. 標準入出力のイメージ

コマンドは「入口1つ（標準入力）」「出口2つ（標準出力・標準エラー）」を持つ。 画面に普通に出るのが標準出力、エラーメッセージが標準エラー。

2. リダイレクト（出力をファイルへ）

# 一覧をファイルへ保存
ls > a/ls-output.txt

# 追記（上書きしない）
echo "追加" >> a/ls-output.txt

# エラーだけを保存
rm not_exist 2> errors.txt

# 標準出力と標準エラーをまとめて保存
somecmd > all.txt 2>&1

3. ファイル表示コマンド

# ファイル中身を表示（連結も可）
cat ls-output.txt

# ページ送りで表示（J:下 / K:上 / Q:終了）
less ls-output.txt

# 先頭・末尾だけ
head -5 ls-output.txt
tail -10 ls-output.txt

4. パイプ（コマンドをつなぐ）

# /bin の一覧をページ送りで見る
ls /bin | less

# "ss" を含む行だけ抽出
ls /bin | grep ss

# "apt" を含まないものだけ
ls /bin | grep -v apt

5. 練習

課題：「/bin にある項目のうち apt を含まないもの」を ~/io-study/non-apt-command.txt に保存せよ。 
# 例解答
mkdir -p ~/io-study
ls /bin | grep -v apt > ~/io-study/non-apt-command.txt
head ~/io-study/non-apt-command.txt

チートシート（超要約）

# 出力→ファイル   >（上書き）,  >>（追記）
# エラー出力       2> file
# 両方まとめる     > file 2>&1
# パイプ           cmd1 | cmd2
# 検索             grep PATTERN [FILE]
